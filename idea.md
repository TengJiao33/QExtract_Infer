### 🚀 项目代号：`QExtract-Infer` 
**(面向长文本信息抽取的 QLoRA 极致融合与极简流式推理微内核)**

**1. 场景特化：抛弃 vLLM (PagedAttention) 的沉重包袱**
*   **业界常规实践**：vLLM 等主流框架是为了“通用聊天（Chat）”设计的，因为多轮对话长度不可测，必须引入极其复杂的 PagedAttention（动态分页显存管理）来防止显存碎片化。但这会带来庞大的页表寻址开销。
*   **你的颠覆点**：你的第一篇顶刊论文做的是**“非结构化评论抽取三元组”**。信息抽取任务（IE）的特点是：**极其庞大的前置文本（Prefill） + 极短的结构化输出（Decode，通常几十个Token）**。在这种极端不对称场景下，动态分页是累赘。你将在 C++ 层面实现一个**“静态连续环形显存池 (Static Ring-Buffer KV-Cache)”**，完全剥离页表映射，将短生成的显存访存时间压榨到物理极限。

**2. 极限压榨：直击 QLoRA 部署的显存带宽墙 (Memory Wall)**
*   **业界常规实践**：标准框架在部署你论文中的 QLoRA 模型时，计算图是割裂的：将底座 INT4 权重反量化为 FP16 $\rightarrow$ 计算底座矩阵乘 $\rightarrow$ 计算 LoRA 的 A 矩阵 $\rightarrow$ 计算 B 矩阵 $\rightarrow$ 结果相加。这会导致 GPU 全局显存被反复读写4次以上，陷入严重的 Memory-Bound（显存带宽瓶颈）。
*   **你的颠覆点**：你将手写一个针对解码期（Decode Phase）的 **`W4A16-LoRA Fused GEMV`（量化底座与微调分支一体化融合算子）**。在 CUDA 线程块内，利用极速的 Shared Memory，**在一个 Kernel 里一次性完成**：底座权重的在线反量化、底座乘法、LoRA 补偿。不产生任何中间张量，将 4 次显存读写压缩为 1 次！

**3. 评测升维：引入“能耗建模”视角的底层基准测试**
*   **业界常规实践**：传统的底层基准测试只看吞吐量（Tokens/s）和延迟（Latency）。
*   **你的颠覆点**：完美联动你第二段科研（无人机物理能耗评估）的思维！结合你对系统能耗的理解，你在测试框架中引入 NVIDIA `pynvml` 库，高频轮询监控 GPU 的瞬时功率。在 README 中，你不仅展示延迟，还独创性地给出 **“Energy per Token (焦耳/Token)”** 的硬件级能效指标，证明你的算子优化不仅提升了速度，还削峰了 GPU 的峰值功耗。

### 🛠️ 详细技术架构与模块拆解

不要试图用 C++ 重写整个大模型加载权重的过程（那是无用功）。采用 **PyTorch + PyBind11 + 纯 CUDA `.cu` 核心算子** 的“外科手术式替换（Monkey Patching）”路线，主打小而精。

#### 模块一：热身与基建 —— 极简 C++ 显存管理器 (Week 1-2)
*   **目标**：用 C++ 的裸指针接管 PyTorch 臃肿的 KV-Cache 张量。
*   **实现细节**：结合业界前沿的 StreamingLLM（注意力汇聚）理论。在 C++ 层面预分配一块大显存池。
    *   永远锁定并保留前 4 个 Token（系统提示词等“汇聚点”）。
    *   将后续分配的显存视为 Ring Buffer（环形缓冲区）。当输入长度超过阈值（如 4096）时，C++ 指针绕回头部，直接覆盖最老旧的 Token。**在 O(1) 显存复杂度下支持你论文中的长文本抽取，绝对不爆显存（OOM）。**

#### 模块二：练手与探秘 —— 常规算子的极速融合 (Week 3-4)
*   **目标**：挑选 Qwen 的 `RMSNorm` 和 `SwiGLU` (MLP激活层)，用纯 CUDA 重写。
*   **实现细节（硬核）**：
    *   摒弃 PyTorch 的分步计算。利用 CUDA 的 `__shfl_down_sync` (Warp级原语) 实现极速归约求方差。
    *   强制使用 `float4` 进行向量化访存（Vectorized Memory Access），一次读 128 bit 数据。将原生 PyTorch 的 3 次显存读写压缩为 **1 次读取 + 1 次写入**。

#### 模块三：核武器登场 —— 动态 QLoRA 融合解码算子 (Week 5-7)
*   **目标**：写一个定制的 CUDA Kernel，专门处理逐字生成阶段的 $Y = X \cdot W_{int4\_base} + X \cdot A \cdot B$。
*   **实现细节（全项目最难、最耀眼的点）**：
    1.  分配 Thread Block 读取输入的向量 $X$。
    2.  异步拷贝（Async Copy）INT4 的底座权重块到 Shared Memory。
    3.  在 GPU 寄存器（Registers）级别直接进行位运算反量化。
    4.  **绝对不要写回显存！** 紧接着在同一个 Kernel 中，将规模很小的 LoRA $A$ 和 $B$ 权重加载，计算 LoRA 分支。
    5.  在寄存器层面合并底座和 LoRA 的结果，再一次性写回全局显存。
*   **精度对齐**：每写完一步，必须用 `torch.allclose(atol=1e-3)` 和官方 PyTorch 结果进行数值误差测试，保证等价性。

#### 模块四：极客级的开源包装 (Week 8)
*   **Nsight Compute (NCU) 抓包**：使用英伟达官方工具，生成你的定制算子与原生 PyTorch 在 **Memory Throughput (显存吞吐量)** 上的对比火焰图。
*   **一键替换**：通过 `pybind11` 打包。在原生 HuggingFace Qwen 代码中，只需要一行代码 `import qextract; qextract.patch_qwen(model)`，就能瞬间把你优化的底层算子替换进去。
